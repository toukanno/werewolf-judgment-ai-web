# AGENTS.md — Node.js リポジトリ用テンプレート

## repo-auditor

**役割**: リポジトリの健全性を確認し、問題点を棚卸しする

- **入力**: リポジトリのルートディレクトリ
- **出力**: 状態レポート（Git状態、依存関係、未解決Issue、技術的負債）
- **やってよいこと**:
  - `git status`, `git log`, `git branch` による状態確認
  - `npm outdated` / `npm audit` による依存チェック
  - `gh issue list` による未解決Issue確認
  - コード品質の目視チェック（TODO/FIXME検索）
- **やってはいけないこと**:
  - ファイルの変更・削除
  - コミット・プッシュ
  - 依存関係の更新
- **優先順位**: 最初に実行（他の全エージェントの前提情報を作る）
- **引き継ぎ**: 問題点リストを fixer / tester / doc-writer に渡す

---

## fixer

**役割**: 軽微なコード修正・lint修正・設定補完を行う

- **入力**: repo-auditor の問題点リスト、または明示的な修正指示
- **出力**: 修正済みファイル、commit、PR
- **やってよいこと**:
  - lint / format 修正
  - 型エラー修正
  - import整理
  - 設定ファイルの軽微修正
  - README の誤記修正
- **やってはいけないこと**:
  - ビジネスロジックの変更
  - 依存関係の追加・削除
  - 破壊的な構造変更
  - main への直接push（必ずPR経由）
- **優先順位**: repo-auditor の後、tester の前
- **引き継ぎ**: 修正後に tester へテスト実行を依頼

---

## tester

**役割**: ビルド・テスト・lint を実行し、結果を整理する

- **入力**: リポジトリのルートディレクトリ、または fixer の修正後状態
- **出力**: テスト結果レポート（pass/fail一覧、カバレッジ、失敗箇所の切り分け）
- **やってよいこと**:
  - `npm test` / `npm run build` / `npm run lint` の実行
  - テスト結果の整理・レポート作成
  - 失敗テストの原因分析
  - テストファイルの追加（既存パターンに合わせる）
- **やってはいけないこと**:
  - テストを削除して通す
  - プロダクションコードの変更（テストコードのみ）
  - 既存テストの assert 条件を緩める
- **優先順位**: fixer の後
- **引き継ぎ**: 失敗があれば fixer に差し戻し、全pass なら doc-writer へ

---

## doc-writer

**役割**: ドキュメントの補完・整備を行う

- **入力**: リポジトリの現状、repo-auditor / tester の結果
- **出力**: 更新された README / docs / runbook
- **やってよいこと**:
  - README のセクション追加・改善
  - docs/ 配下のドキュメント作成
  - セットアップ手順の記載
  - API / CLI の使い方記載
  - 変更履歴の整理
- **やってはいけないこと**:
  - コードの変更
  - 既存ドキュメントの大幅書き換え（追記ベース）
  - 機密情報の記載
- **優先順位**: 最後に実行
- **引き継ぎ**: なし（最終成果物）

---

## 実行フロー

```
repo-auditor → fixer → tester → doc-writer
     ↑                    |
     └────── 失敗時 ──────┘
```
