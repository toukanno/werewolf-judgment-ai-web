# ゲーム設計書

## 役職一覧

### MVP 役職（最初に実装）

| 役職 | 陣営 | 能力 | 勝利条件 |
|---|---|---|---|
| 村人 | 村人陣営 | なし | 人狼を全員処刑する |
| 占い師 | 村人陣営 | 夜に1人を占い、人狼かどうか知る | 人狼を全員処刑する |
| 騎士 | 村人陣営 | 夜に1人を護衛し、襲撃から守る（連続護衛不可） | 人狼を全員処刑する |
| 人狼 | 人狼陣営 | 夜に1人を襲撃して殺害する | 村人の数 ≤ 人狼の数 |

### 拡張役職（将来実装）

| 役職 | 陣営 | 能力 |
|---|---|---|
| 霊能者 | 村人陣営 | 処刑された人が人狼だったか知る |
| 狂人 | 人狼陣営 | 能力なし。占い結果は「村人」と出る |
| 妖狐 | 第三陣営 | 襲撃されても死なない。占われると死ぬ。生存勝利 |
| ハンター | 村人陣営 | 死亡時に1人を道連れにする |

## ルール概要

- プレイヤー数: 5〜9人（うち1人が人間プレイヤー）
- 役職はゲーム開始時にランダムに割り振り
- 昼と夜を繰り返す
- 1日目の昼は処刑なし（議論のみ）

## ゲーム進行

```
[ゲーム開始]
  ↓
[役職配布] → プレイヤーに役職を通知
  ↓
[1日目・昼] → 議論のみ（処刑なし）
  ↓
[1日目・夜] → 占い師: 占い / 人狼: 襲撃 / 騎士: 護衛
  ↓
[2日目・昼] → 襲撃結果発表 → 議論 → 投票 → 処刑
  ↓
[勝敗判定] → 決着していなければ夜へ
  ↓
[2日目・夜] → 各役職の能力行使
  ↓
（繰り返し）
```

## 昼フェーズ

1. 前夜の襲撃結果を発表（「○○さんが無残な姿で発見されました」）
2. 議論（AI プレイヤーが順番に発言）
3. 投票（全員が処刑対象を選ぶ）
4. 最多票の人を処刑（同数の場合はランダム）
5. 勝敗判定

### 議論の流れ
- AI プレイヤーは1人ずつ発言する
- 発言内容は役職・状況に応じて AI が生成
- 人間プレイヤーも発言できる（テキスト入力）
- 議論は 1 ラウンド（全員が1回発言）で終了

### 投票処理
- 全プレイヤーが処刑対象を1人選ぶ
- 自分への投票は不可
- 最多票で処刑（同数ならランダム）
- 投票先は公開

## 夜フェーズ

1. 占い師 → 占い対象を選択（結果: 人狼 or 村人）
2. 騎士 → 護衛対象を選択（前夜と同じ人は不可）
3. 人狼 → 襲撃対象を選択
4. 襲撃判定（護衛されていれば失敗）

### 夜の処理順序
1. 占い → 結果を占い師に通知
2. 護衛 → 護衛対象を記録
3. 襲撃 → 護衛対象と一致すれば「襲撃失敗」

## 勝敗条件

| 条件 | 勝者 |
|---|---|
| 人狼が全員処刑された | 村人陣営の勝利 |
| 生存者の中で 人狼の数 ≥ 村人陣営の数 | 人狼陣営の勝利 |

## AI 対戦の設計方針

### API 構成
- OpenRouter API を使用
- ユーザーが自分の API キーを入力
- モデル: `anthropic/claude-sonnet-4` をデフォルト
- API キーはローカル（localStorage）に保存、サーバーには送信しない

### AI プレイヤーの思考
- 各 AI プレイヤーに「性格」「プレイスタイル」を設定
- プロンプトに以下を含める:
  - 自分の役職
  - 生存者リスト
  - これまでの議論ログ
  - 投票履歴
  - 占い結果（占い師の場合）
- AI は JSON 形式で応答を返す

### プロンプト設計（概要）

```
あなたは人狼ゲームのプレイヤーです。
名前: {name}
役職: {role}
性格: {personality}

現在の状況:
- 生存者: {alive_players}
- 本日の議論: {discussion_log}
- 過去の投票結果: {vote_history}

以下の形式で応答してください:
{
  "statement": "あなたの発言",
  "reasoning": "内部的な思考（非公開）"
}
```

## API キー入力 UI 方針

- ロビー画面に API キー入力欄を配置
- 入力したキーは localStorage に保存
- 「テスト接続」ボタンで疎通確認
- キーが未入力でもモック AI で遊べるモードを用意
